# SmartAgent

SmartAgent 是一个面向多 Agent 协作的智能对话项目，围绕“输入理解 - 状态调整 - 工具决策 - 回复生成 - 记忆存储”的完整链路构建。

## 项目目标

项目的核心目标是把用户输入转化为更自然、更连贯、带有个性化风格的回复，同时保留短期对话上下文和长期记忆能力。

## 当前架构概览

当前代码已从旧的流程式模块，迁移为以 `src/agents/` 为核心的职责拆分结构：

- 输入处理 Agent：负责 query 判断、扩写、情绪补种与记忆召回
- 状态调整 Agent：负责人格、语气、情绪联动调整
- 工具 Agent：负责工具决策与工具执行
- 回复 Agent：负责最终回复生成与对话记忆写入
- 上下文与仓储层：负责 Prompt 拼装、Redis / MongoDB / Milvus 数据访问

## 核心能力

- 用户输入预处理与扩写
- 短期记忆与长期记忆召回
- 人格、语气、情绪等上下文建模
- 工具调用结果融合
- 基于上下文生成最终回复
- 对话记录与记忆持久化

## 主要目录

- `src/agents/`：核心 Agent 实现，包括输入、状态、工具、回复四类 Agent
- `src/core/`：上下文对象与 Prompt 构建核心逻辑
- `src/InputProcess/`：输入处理相关基础组件，如扩写、情绪分析、记忆召回与压缩
- `src/Tools/`：LLM 客户端、外部工具封装与工具注册机制
- `src/database/`：数据库客户端、仓储与记忆相关功能
- `src/prompts/`：按职责拆分的 Prompt 模板

## 当前处理流程

1. 接收用户输入并初始化 `Context`
2. 输入处理 Agent 判断是否需要扩写与记忆召回
3. 补充情绪信息并更新主线记忆
4. 状态 Agent 调整人格与语气配置
5. 工具 Agent 判断是否调用外部工具并写回结果
6. `Context.build_prompt()` 拼装最终上下文
7. 回复 Agent 调用大模型生成回复
8. 将本轮对话写入短期与长期记忆

## 依赖的存储与服务

项目代码中使用了多种外部服务或组件，常见包括：

- Redis：短期记忆
- Milvus：向量记忆
- MongoDB：用户画像与人格状态配置
- 外部大模型服务：用于回复生成、分析与向量化
